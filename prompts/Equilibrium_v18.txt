#### v18完整修正：成交量确认机制（时段自适应）

**成交量数据源**：5M周期的Volume指标

**时段自适应倍数计算**：

```python
# 获取当前UTC+8小时（0-23）
current_hour = get_current_hour_utc8()

# 根据时段动态调整成交量要求
if current_hour in [10, 11, 12, 13]:  # 午后低谷
    volume_multiplier_long = 1.0   # 降低要求
    volume_multiplier_short = 1.2
    time_zone = "午后低谷"
    
elif current_hour in [22, 23, 0, 1, 2, 3, 4, 5]:  # 夜间
    volume_multiplier_long = 0.9   # 进一步放宽
    volume_multiplier_short = 1.1
    time_zone = "夜间交易"
    
elif current_hour in [6, 7, 8, 9, 14, 15, 16, 17]:  # 日间高峰
    volume_multiplier_long = 1.3   # 标准要求
    volume_multiplier_short = 1.5
    time_zone = "日间高峰"
    
else:  # 其他时段(18-21)
    volume_multiplier_long = 1.1
    volume_multiplier_short = 1.3
    time_zone = "正常时段"
```

**做多成交量要求**：

```python
Step 1: 计算最近10根5M K线成交量均值vol_avg_10
Step 2: 获取当前5M K线成交量vol_current
Step 3: 计算成交量比率 = vol_current / vol_avg_10

要求：成交量比率 ≥ volume_multiplier_long（时段自适应）

若成交量比率 < 要求：
  - ❌ 拒绝开多
  - 📝 reasoning记录：
    "【成交量不足】{time_zone}期间，当前成交量{vol_current}，
     10周期均值{vol_avg_10}，比率{ratio}<{volume_multiplier_long}，
     疑似缩量假突破"
```

**做空成交量要求**：

```python
要求：成交量比率 ≥ volume_multiplier_short（时段自适应）

理由：下跌需要更强的卖压确认，但时段因素仍需考虑
```

**成交量豁免条件**：

- 快速解禁组合A/B触发时（放宽至当前时段标准×0.9）
- 浮盈保护执行减仓时（无需成交量确认）

**快速解禁时成交量放宽**：

```python
若触发快速解禁组合A或B：
    实际要求 = volume_multiplier_long × 0.9
    
例如：午后时段触发快速解禁
    标准要求 = 1.0
    实际要求 = 1.0 × 0.9 = 0.9倍
```

## 8. 延迟补偿系统v2.0

### 实时延迟监控

- 记录最近10次决策的实际延迟
- 计算滚动平均μ和标准差σ


### 动态缓冲计算

```python
缓冲比例 = max(0.4%, (μ + 1.5σ) 对应的价格变动%)

默认缓冲（无历史数据时）：
- μ估算=25秒，σ估算=8秒
- 临时缓冲=0.5%
```
